# 系统架构

## 整体架构

RDK550P-MOSS 是一个运行在 D-Robotics RDK X5 上的桌面 AI 机器人系统，由三层组成：

### 1. 硬件层

```
┌─────────────────────────────────────┐
│           RDK X5 Board              │
│  ┌─────────┐  ┌──────────────────┐  │
│  │ ARM A55 │  │ BPU (8 TOPS)     │  │
│  │ 4-core  │  │ 视觉推理加速     │  │
│  └─────────┘  └──────────────────┘  │
│                                     │
│  MIPI CSI ──→ IMX477 摄像头         │
│  PWM x2   ──→ PTK7465 舵机 (云台)   │
│  USB      ──→ 音频模块 (麦克风+喇叭)│
└─────────────────────────────────────┘
```

### 2. 感知与执行层（Voice Assistant）

语音助手是常驻守护进程，负责所有实时感知：

```
音频输入 (arecord)
    │
    ▼
┌─────────────┐
│ Silero VAD  │  ← 持续监听，检测语音段
└──────┬──────┘
       │ 检测到语音
       ▼
┌─────────────┐
│ 唤醒词匹配   │  ← ASR 文本中包含唤醒词？
└──────┬──────┘
       │ 匹配成功
       ▼
┌─────────────┐     ┌──────────────┐
│ SenseVoice  │────→│ Speaker ID   │  ← 识别"谁在说话"
│ (ASR)       │     │ (3DSpeaker)  │
└──────┬──────┘     └──────────────┘
       │ 文字指令
       ▼
┌─────────────┐
│ OpenClaw    │  ← LLM 理解意图、生成回复
│ Agent CLI   │
└──────┬──────┘
       │ 文字回复
       ▼
┌─────────────┐     ┌──────────────┐
│ edge-tts    │────→│ aplay        │  ← 语音播放
│ (TTS)       │     │ (ALSA)       │
└─────────────┘     └──────────────┘
```

关键设计决策：
- **录音与播放互斥** — USB 声卡不支持全双工，播放时暂停录音
- **arecord 替代 sherpa_onnx.Alsa** — 后者在此硬件上不可靠
- **唤醒词用 ASR 文本匹配** — 而非专用 KWS 模型（更灵活，部署更简单）

### 3. 智能层（OpenClaw）

OpenClaw 是 AI 大脑，提供：

- **LLM 对话** — 接入 Claude/GPT 等模型
- **记忆系统** — MEMORY.md（长期）+ memory/ 日志（短期）
- **定时任务** — Cron + 心跳机制
- **工具调用** — 舵机控制、拍照、TTS 等
- **人格定义** — SOUL.md 定义机器人性格

```
┌──────────────────────────────────────┐
│           OpenClaw Gateway           │
│                                      │
│  ┌────────────┐  ┌───────────────┐   │
│  │ Agent      │  │ Memory        │   │
│  │ Runtime    │  │ (SOUL/USER/   │   │
│  │            │  │  MEMORY.md)   │   │
│  └─────┬──────┘  └───────────────┘   │
│        │                             │
│  ┌─────▼──────┐  ┌───────────────┐   │
│  │ Tool       │  │ Cron / HB     │   │
│  │ Execution  │  │ Scheduler     │   │
│  └────────────┘  └───────────────┘   │
└──────────────────────────────────────┘
```

## 进程模型

```
systemd
├── openclaw-gateway.service    # OpenClaw 网关（常驻）
├── voice-assistant.service     # 语音助手（常驻，崩溃自动重启）
└── ...
```

两个服务独立运行。语音助手通过 `openclaw agent` CLI 与 OpenClaw 通信，不直接调用 LLM API。

## 数据流

1. **语音输入** → arecord → VAD → ASR → 文字
2. **意图理解** → openclaw agent → LLM → 文字回复
3. **语音输出** → edge-tts → ffmpeg → aplay
4. **物理动作** → Python PWM 控制 → 舵机运动
5. **视觉输入** → libsrcampy → JPEG → 文件/分析
